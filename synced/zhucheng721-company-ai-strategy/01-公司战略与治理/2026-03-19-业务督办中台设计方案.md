# 业务督办中台设计方案（最终版）

**日期：** 2026-03-19
**版本：** V1 最终版（合并 Claude + Codex 方案）
**适用对象：** Rita + 直接管理层 + 小龙虾机器人
**目标：** 解决"任务交代出去了，但没有持续跟进、没有结论沉淀、没有日报汇总"的管理断层

---

## 一、为什么要做这个中台

当前公司内部最大的管理问题，不是"没人做事"，而是：

1. Rita 交代了事情，员工也回复了，但后续没有持续进展回传
2. 事情处理中途卡住，没有统一提醒和催办机制
3. 做完了也没有形成正式结论，后面再遇到类似问题还要重新问
4. 每天发生了什么、推进了什么、还缺什么，没有一份统一业务摘要

结果就是：

- Rita 不知道哪些任务还在办
- Rita 不知道哪些任务已经形成明确结论
- 员工做了事，但公司没有把这些结果变成组织记忆

所以要搭的不是一张表，而是一套最小可用的闭环机制：

`交办 → 跟进 → 结案 → 日报 → 结论沉淀`

---

## 二、这套系统的定位

**正式命名：业务督办中台**

它不是 OA、CRM、复杂项目管理系统、全员手工填表系统。

它是：

- Rita 的任务分发与跟进中心
- 员工的进展回报入口
- 公司结论沉淀与日报汇总中心

核心原则：

**员工不填表、不用记格式。员工在群里用自然语言说话，小龙虾负责理解、写表、汇总、提醒。**

**员工群人员：** Rita、申鸿龙、梦琪、客服主管、运营相关负责人、小龙虾

### 小龙虾需要 @吗？——混合模式

| 场景 | 是否需要 @ | 原因 |
|------|-----------|------|
| **Rita 布置任务** | 不需要 @ | 小龙虾自动识别"让XX做..."这类指令 |
| **员工汇报进展** | 建议 @ | "@小龙虾 A客户签约了500单"——避免闲聊被误识别 |
| **员工报业务数据** | 建议 @ | "@小龙虾 B线今天1200单，新增2家" |
| **闲聊/讨论** | — | 小龙虾不响应 |

员工只需养成一个习惯：**汇报时 @小龙虾**，其他时候正常聊天不受影响。

| **需持续跟踪** | 是否需要后续=是 | Rita 长线任务管理 |

## 六、小龙虾的自然语言交互方式

### 核心原则：像跟人说话一样，不用背格式

小龙虾不是"表单填写工具"，而是一个能听懂人话的管理助手。员工和 Rita 用自然语言说话，小龙虾自动理解并结构化。

### 6.1 Rita 交代任务时

**Rita 在群里说：**
> "小张，跟进一下 A 客户的退货对接，这周五之前要有结论"

**小龙虾自动做的：**
1. 识别这是一条任务（关键词：跟进/处理/完成/对接/确认/搞定...）
2. 提取结构化信息：
   - 责任人：小张
   - 任务：跟进 A 客户退货对接
   - 截止日：本周五（2026-03-21）
   - 业务线：B-退货
   - 优先级：🟡 重要
3. 写入飞书表 3（任务督办表）
4. 在群里回复确认：
   > "✅ 已记录任务：**跟进 A 客户退货对接**
   > 责任人：小张 | 截止：3月21日(周五) | 优先级：🟡重要
   > 我会在到期前提醒。"

### 6.2 员工汇报进展时

**员工说：**
> "@小龙虾 A 客户退货对接已经谈了，他们要先试用一周"

**小龙虾自动做的：**
1. 匹配到现有任务（A 客户退货对接）
2. 更新任务状态：进行中
3. 更新最新进展："客户要先试用一周"
4. 回复确认：
   > "✅ 已更新任务进展：A 客户退货对接
   > 状态：进行中 → 客户试用一周
   > 预计下周可出结论，届时我会提醒你更新。"

### 6.3 员工汇报完成时

**员工说：**
> "@小龙虾 A 客户退货对接搞定了，签了合同，首批 500 单"

**小龙虾自动做的：**
1. 更新表 3 状态：已完成
2. 填入结论："签约成功，首批 500 单"
3. **同时写入表 5（结论与决策表）**：沉淀为正式结论
4. 回复确认 + 通知 Rita：
   > "🎉 任务完成：A 客户退货对接
   > 结论：签约成功，首批 500 单
   > 已沉淀到结论库。@Rita 这个任务已闭环。"

### 6.4 员工报业务数据时

**员工说：**
> "@小龙虾 今天 B 线退货 1200 单，新增客户 2 家，跟进中 5 家"

**小龙虾自动做的：**
1. 写入表 4（业务数据日报表）
2. 回复确认：
   > "✅ B线今日数据已记录：1200单 | 新增2家 | 跟进中5家"

### 6.5 信息不完整怎么办

如果小龙虾识别到是任务但信息不够，**主动追问**，不乱猜：

> "收到任务，但我还需要确认几个信息：
> 1. 截止日期是什么时候？
> 2. 这个归哪条业务线？
> 请补充一下，我再入表。"

## 七、小龙虾自动化规则

### 每日自动推送

| 时间 | 推什么 | 推给谁 |
|------|-------|-------|
| **早 8:30** | "今日待办清单"——筛出截止日≤今天且未完成的任务，按员工分组 | 员工工作群（全员可见） |
| **晚 18:00** | 催办提醒——超期未完成的任务，@责任人 | 员工工作群 |
| **晚 20:00** | "今日业务日报"——汇总当天各线数据 + 任务完成情况 + 新增结论 | 管理群（推给 Rita） |

### 自动化触发规则

| 触发条件 | 动作 |
|---------|------|
| 任务状态改为"待确认" | 通知 Rita："XX 任务已完成，等你确认" |
| 任务截止日=明天 & 状态≠已完成 | 通知责任人："XX 任务明天到期" |
| 任务超期 2 天 & 状态≠已完成 | 通知 Rita + 责任人："XX 任务已超期 2 天" |
| 任务超期 5 天 | 升级提醒 Rita："XX 任务严重超期，建议处理" |
| 某员工连续 2 天无任何任务更新 | 提醒 Rita："XX 今天/昨天没有更新任何任务进展" |
| 某业务线当天无数据填报 | 提醒对应负责人："今天 X 线数据还没报" |
| 任务结案且有明确结论 | 自动写入表 5（结论与决策表） |

### 周一周报

每周一 09:00 推送到管理群：
- 上周完成任务
- 本周仍未闭环任务
- 各负责人积压情况
- 各业务线核心风险
- 本周需 Rita 重点盯的事项
- 上周新增结论汇总

## 九、技术实现架构

### 9.1 飞书 API 自动写入

小龙虾（基于 Claude Sonnet 4.6）本身能理解语言，但需要"手"来写飞书表。

**推荐组合：A 为主 + B 补充**

| 方案 | 原理 | 难度 | 用途 |
|------|------|------|------|
| **A. 飞书机器人 + 多维表格 API** | 小龙虾收到消息→调用飞书开放平台 API→写入多维表格 | 中等 | ⭐ 主方案：任务识别+写入 |
| **B. 飞书自动化（无代码）** | 用飞书自带的自动化功能，设置触发规则 | 低 | 补充：定时推送+催办提醒 |

### 9.2 需要确认的前提

| 确认项 | 说明 | Rita 需要做什么 |
|-------|------|---------------|
| 小龙虾的机器人类型 | 是"自建应用机器人"还是"飞书官方 AI 助手"？ | 告诉我你在飞书后台怎么设置的小龙虾 |

| 小龙虾是否支持调用外部 API | 不同版本的飞书 AI 机器人能力不同 | 需要确认 |

## 十、小龙虾 Prompt 设计要点

小龙虾的系统 Prompt 需要包含以下自然语言理解能力：

### 任务识别规则

```
当群里有人说了包含以下关键词的话时，识别为"任务"：
- 跟进/处理/确认/完成/搞定/对接/安排/催/查/核实/回复
- "让XX做..."/"XX你去..."/"这个事情XX负责"
- "什么时候之前..."/"周X之前..."/"今天必须..."

识别后提取：
- 责任人（谁去做）
- 任务内容（做什么）
- 截止时间（什么时候前）
- 优先级（根据语气和时间推断）
- 业务线（根据内容判断）
```

### 进展识别规则

```
当有人说了包含以下关键词的话时，识别为"进展更新"：
- "XX搞定了"/"XX完成了"/"XX做好了"
- "XX还在等..."/"XX在处理中"
- "XX签约了"/"XX谈成了"/"XX拒绝了"

匹配到已有任务后更新状态和进展。
```

### 数据识别规则

```
当有人说了包含数字+业务关键词时，识别为"业务数据"：
- "今天XX单"/"新增X个客户"/"跟进中X家"
- "收入XX元"/"签了X单"

提取后写入业务数据日报表。
```

### 结论识别规则

```
当任务结案且包含明确结论时，同步写入结论与决策表：
- "最终定了..."/"结论是..."/"签约了..."/"决定不做了..."
- 任何 Rita 说的"就这么定了"/"按这个来"

提取后写入表 5，标注决策人和生效日期。
```

## 十三、当前版本边界

本方案是 V1 最小可用版，先解决四件事：

1. Rita 交办的事情不会丢
2. 员工进展会回传
3. 公司结论能沉淀下来
4. 每天有一份结构化的业务摘要

当前不做的：复杂绩效考核、全自动流程审批、复杂 CRM、财务审批流、全员精细化人效统计。V1 跑顺后再考虑。

## 十四、未来可扩展方向

1. 任务自动关联客户台账

3. 和飞书日历联动截止日期提醒
4. 和内容中台联动，任务自动生成宣传素材需求

## 十五、最重要的一句话

这套系统不是为了让员工多做事，而是为了让公司形成最基础的管理闭环：

**用自然语言说话，小龙虾帮你管。事情发出去了，就必须看得到推进、看得到结论、看得到未完成项。**
