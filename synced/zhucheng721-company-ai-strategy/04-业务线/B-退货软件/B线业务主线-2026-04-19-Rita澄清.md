# B 线业务主线图（Rita 2026-04-19 澄清版）

## 核心结论：B 线是"软件 + 仓储 + 渠道"三位一体

不是单一软件生意，是围绕"退货"这个场景的**三条能力同时输出**：

```
┌────────────────────────────────────────────────┐
│           B 线 · 以退货为轴心的三位一体            │
├────────────────────────────────────────────────┤
│                                                │
│   软件层 (SaaS)                                 │
│   ├─ 给快递老板的退货软件（提高派费）              │
│   ├─ 给电商老板的退货 WMS（内销物流软件，丝滑处理） │
│                                                │
│   仓储层 (云仓)                                  │
│   └─ 自己的云仓承载电商的退货处理（委外服务）       │
│                                                │
│   渠道层 (经纪)                                  │
│   └─ 帮大电商找快递站点承载退货 + 返点             │
│                                                │
└────────────────────────────────────────────────┘
```

---

## 4 种客户 × 我们提供什么

| 客户类型 | 客户痛点 | 我们给什么 | 盈利模式 |
|---------|---------|----------|---------|
| **快递老板（加盟圆通/中通/韵达等）** | 退货件=集派=派费低 | **退货软件**：通过软件处理退货件，提高派费 | 软件服务费 + 派费差额 |
| **大电商（需要软件）** | 退货业务乱，想自动化 | **退货 WMS 软件**：内销退货物流软件，丝滑处理 | SaaS 订阅费 |
| **大电商（需要渠道）** | 退货件不知道送到哪、没议价权 | **快递站点匹配 + 返点**：帮他们找快递站点承载退货，我们收返点给他们一部分 | 渠道返点分成 |
| **电商公司（想委外云仓）** | 自己处理退货太耗精力 | **我们的云仓承载**：从仓储到退货处理全托管 | 仓储费 + 退货处理费 |

## 关键业务红线（继续有效）

### 🔴 快递老板端
- **不提"软件/系统/技术/改地址"**
- 只讲"赚钱 / 每单多赚 0.4-0.5 元 / 月增 1.5 万+"
- 详见 `B线话术体系-2026-03-31定稿.md`

### 🟢 电商老板端
- 可以**正常讲产品功能**（策略配置、羊毛党识别、自动退款、ABCD 质检分级、CD 产品赋能）
- 这是**可以公开展示的 WMS 产品**

### 🟢 对需要云仓委外的电商
- 讲"托管、丝滑、省精力、专业团队"
- 讲真实案例和数据

## 内容方向对照（给 content-planner / copywriter 用）

### 对【快递老板】讲什么

- ❌ "我们的软件能帮你改退货地址、提高派费"（违反红线）
- ✅ "做快递这几年，退货件越来越多，派费却一直被压。别家老板都在找新办法，你还在等吗？"
- ✅ "快递涨价了，为什么你还是没多赚钱？"
- ✅ "派费如果不跟着涨，加盟商今年最该盯什么"

### 对【大电商需要软件】讲什么

- ✅ "退货业务占了运营团队 40% 精力？WMS 能降到 5%"
- ✅ "羊毛党识别 + 自动分级 + 一键退款，退货流程真正丝滑"
- ✅ 产品功能可以正面讲

### 对【大电商需要渠道】讲什么

- ✅ "退货件怎么发才更省？我们对接 300+ 快递站点，还给你返点"
- ✅ "一单退货成本降 30%，靠的不是砍运费，是换承运结构"

### 对【电商想委外云仓】讲什么

- ✅ "退货不是小事，但也不该占你最好的精力。让专业云仓接手"
- ✅ "从收货到分级到二次上架，我们全包"
- ✅ "已经在托管 N 家电商客户，退货处理时长从 X 天压到 Y 天"（数据后续补）

## 系统"在建"时期的客户应对话术（2026-04-19 澄清追加）

**背景**：公司整体推广系统+业务架构正在升级，尚未完成转换。但软件主体可用，云仓实体和渠道能力已存在。

**统一口径（严禁违反）**：

- ❌ 禁止说"还没做好" / "推广系统没打通" / "框架在转换" / "还在开发"
- ❌ 禁止夸大说"全部做好了"
- ✅ 统一说"**软件主体已经在跑，正在做面向'快递老板自营云仓'场景的升级**"
- ✅ "**首批客户政策**"话术：你现在进来，享受早期客户优惠 + 按你实际业务配置
- ✅ 强调 "2-3 周配置期 + 免费培训 + 30 天试用" 给客户安全感

**4 种应答场景**（详见 `docs/approval/drafts/L-20260419-001-跟进话术-v2.md` 对应段落）：

1. 客户没问 → 不主动说
2. 客户问成熟度 → "软件主体成熟 + 升级为您这种场景做 + 首批客户政策"
3. 客户问上线时间 → "主体随时用 + 2-3 周配置期 + 一对一实施"
4. 客户问整体完整度 → "三块能力（软件/云仓/渠道）都有，正在打包整合成一站式 SOP"
