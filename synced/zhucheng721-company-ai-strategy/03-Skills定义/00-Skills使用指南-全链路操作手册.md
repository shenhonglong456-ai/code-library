# 乾江盛世 AI Skills 全链路操作手册

**版本：** V1.0
**更新：** 2026-04-02
**用途：** 在Claude Code中输入对应指令，系统自动按流程执行全链路闭环

---

## 总览：6条全链路流水线

```
流水线A：情报链路    情报采集 → 深挖 → 竞品 → 舆情 → 需求挖掘
流水线B：内容链路    情报 → 内容规划 → 写文案 → 生图/视频 → 审核
流水线C：分发链路    内容成品 → 平台适配 → 排期 → 发布 → 数据回收
流水线D：转化链路    线索收集 → 自动回复 → CRM管理 → 打包方案 → 报价
流水线E：开发链路    需求分析 → PRD → 架构 → 开发 → 审计 → 测试 → 文档
流水线F：运营链路    数据监控 → 任务督办 → 周报 → 客户上线 → 培训
```

---

## 流水线A：情报链路

### 自动模式（每天07:30 crontab自动执行）
无需操作，系统自动：扫描7线200+关键词 → 五维分析 → 重要等级标注 → 飞书推送

### 手动模式

**1. 查看今日情报**
```
"看看今天的情报简报"
→ 读取 02-市场调研/YYYY-MM-DD-每日情报简报.md
```

**2. 深挖某条情报**
```
"深挖第3条" 或 "深挖这个URL：https://xxx"
→ deep-crawler：Firecrawl全文抓取 → 五维分析 → 存档06-知识库/ → 推飞书
```

**3. 看竞品动态**
```
"跑一次竞品扫描" 或 "查一下聚水潭最近有什么动作"
→ competitor-tracker：Tavily+Perplexity搜索 → Claude汇总 → 竞品周报
```

**4. 社交舆情监控**
```
"看看小红书上退货相关最近讨论什么"
→ social-monitor：agent-reach/web-access搜索 → 提取客户原声+热点
```

**5. 需求挖掘（重要！）**
```
"去小红书挖一下小程序的需求" 或 "看看快递老板在评论区说什么"
→ social-monitor（需求挖掘模式）：搜关键词 → 深入评论区 → 提取需求信号 → 汇总共性 → 输出需求挖掘报告
```

## 流水线B：内容链路

### 全自动模式
```
"根据今天的情报，规划本周内容并写出来"
→ 自动串联：
  1. content-planner 读取今日情报 → 输出内容日历
  2. copywriter 按日历逐条写文案 → 输出内容成品
  3. xhs-image-gen 为小红书内容配图 → 输出图片组
  4. video-producer 为抖音内容做视频 → 输出视频
  5. style-manager 全程管控风格一致性
```

### 分步模式

**1. 规划内容**
```
"根据今天的情报规划内容"
→ content-planner：读情报 → 按6阶段+平台矩阵 → 输出内容日历
```

**2. 写文案**
```
"写一篇小红书文案，B线退货，痛点共鸣型"
→ copywriter：调用B线话术体系 + style-manager风格模板 → 输出成品文案+配图建议+标签
```

**3. 生成配图**
```
"给这篇文案配小红书信息图，5张"
→ xhs-image-gen：选风格(bold/notion/warm) → baoyu-xhs-images生成 → 输出图片组
```

**4. 制作视频**
```
"把这个脚本做成抖音视频，用Seedance 2.0"
→ video-producer：
  - 新画布(change_project) → Seedance 2.0生成
  - 多镜头在同一画布
  - CosyVoice配音(longshu)
  - 输出成品视频
```

## 流水线C：分发链路

### 全自动模式
```
"把今天的内容发出去"
→ 自动串联：
  1. platform-adapter 将内容适配各平台格式
  2. schedule-manager 按排期分配账号+时间
  3. publisher-xhs / publisher-wechat / publisher-douyin 执行发布
```

### 分步模式

**1. 适配格式**
```
"把这篇文案适配成小红书+抖音+公众号三个版本"
→ platform-adapter：一份内容 → 三个平台版本（标题/长度/图片/标签各自适配）
```

**2. 查看排期**
```
"看看今天该发什么"
→ schedule-manager：展示今日发布清单（时间×平台×账号×内容）
```

**3. 发布**
```
"发布到小红书" 或 "发布到公众号"
→ publisher-xhs / publisher-wechat / publisher-douyin：执行发布 → 记录状态
```


## 流水线E：开发链路（最重要的闭环）

### 一句话启动全链路
```
"我要开发一个快递网点AI管家小程序"
```

系统自动按以下顺序执行，每步完成后自动进入下一步：

```
Step 1: requirement-analyzer
  "先帮我分析需求"
  → 6个维度追问 → 输出需求分析文档
  → 完成后自动说："需求分析完成，开始生成PRD"

Step 2: prd-generator
  → 三视图PRD（老板/开发/客户）
  → 用户故事+验收标准+工期预估
  → 完成后自动说："PRD完成，开始设计技术方案"

Step 3: architecture-designer
  → 技术选型（微信云开发+AI）
  → 数据库设计+API设计+系统架构图
  → 完成后自动说："技术方案完成，开始编码"

Step 4: dev-workflow
  → 按PRD用户故事逐个实现
  → 每个功能完成后自动跑单元测试
  → git commit 每个功能点
  → 完成后自动说："编码完成，提交代码审查"

Step 5: code-audit
  → 安全/正确性/性能/风格 全维度审查
  → 输出审查报告（🔴必须修/🟡建议修/⚪可选）
  → 有🔴则回到Step 4修改后重审
  → 完成后自动说："审查通过，开始冒烟测试"

Step 6: 冒烟测试（集成在dev-workflow中）
  → 全链路功能验证
  → 截图记录每个页面/功能的测试结果
  → 发现bug → 自动修复 → 重新测试
  → 完成后自动说："测试通过，生成文档"

Step 7: doc-generator
  → API文档 + 部署文档 + 用户手册 + 更新日志
  → 完成后自动说："文档生成完成，整个开发流程闭环"
```

### 关键规则
- **每步自动进入下一步**，不需要手动触发
- **每步完成后自动记忆和更新文档**（进度/变更/决策）
- **发现bug自动修复**，不需要手动指定
- **冒烟测试自动截图**，存入测试报告
- **全程git commit**，每个功能点一个commit

### 手动干预点
只有以下情况需要你确认：
- PRD中的业务决策（优先级/取舍）
- 报价/定价相关
- 发布到生产环境

## 复盘与自我修复机制

### 每个新Skill/Agent建完后的标准复盘流程

```
"复盘一下刚建的XXX Skill"
→ 自动执行：
  1. 读取SKILL.md定义
  2. 找一个真实场景做demo测试
  3. 对比：实际输出 vs 预期输出
  4. 发现差异 → 自动修改SKILL.md或脚本
  5. 再跑一次验证
  6. 通过 → 标记为"已验证"
  7. 未通过 → 继续修复直到通过
```

### 全链路回归测试
```
"全链路测试一遍"
→ 从情报→内容→发布→转化 跑一遍最小闭环
→ 每个节点检查输入/输出是否正确
→ 发现断点 → 定位哪个Skill出问题 → 修复
```

### 持续更新机制
- 每次对话中修改了Skill → 自动更新SKILL.md
- 新增业务线/话术/规则 → 自动更新company-context-manager
- 发现Bug → 修复后更新到对应Skill的"已知问题"部分
- Rita的反馈/纠正 → 存入记忆系统 + 更新相关Skill

## 流水线G：短剧内容链路

### 制作一集短剧
```
"做一集短剧：快递退货老板的一天，3D CG写实风格"
→ Zopia自动执行：
  1. 生成剧本（角色+场景+对话）
  2. 生成角色设定图
  3. 生成分镜关键帧
  4. 生成每个镜头的视频
  5. 下载全部素材
  6. （手动）FFmpeg拼接+加字幕
```

### 短剧系列规划
```
"规划10集退货行业短剧系列"
→ 每集一个目标群体的日常痛点
→ 每集软植入一个公司产品/服务
→ 不同集可以用不同风格（吉卜力/3D CG/日韩动漫等）
```

### Zopia使用
```bash
# 创建项目
zopia create-base "项目名"

# 设置风格
zopia set-style base_ID --style realistic_3d_cg  # 或 anime_ghibli/pixar_3d_cartoon等

# 发送创作指令
zopia send base_ID "一句话描述你的短剧"

# 查看进度
zopia status base_ID

# 下载成品
zopia download base_ID ~/Downloads/项目名/
```

## ⚠️ 风格选择规则（最高优先级）

**任何内容（图片/视频/信息图）生成前，必须先问Rita选择风格。**

```
❌ 错误："我直接用bold风格给你生成了"
✅ 正确："这次内容你想用什么风格？可选：bold/糖果系/notion/吉卜力/3D CG..."
```

不同账号固定不同风格，由style-manager统一管理。

## 注意事项

1. **B线快递老板端所有内容不提软件** — 这是红线
2. **LibTV使用规则** — 文生视频用Seedance 2.0，生图用Gemini 3.1 Pro，每个独立视频新画布
3. **多工具冗余抓取** — A工具失败自动切B，不能因为一个工具挂了就停
4. **竞品 = 任何能替代我们功能的方案** — 不只看直接竞品
5. **评论区需求挖掘是标准调研方法** — 每条业务线每周至少做一次
6. **0.1%不懂就问Rita** — 不确定的业务逻辑必须确认
